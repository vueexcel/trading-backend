Global site map for MVP
Below is the recommended page inventory (what pages exist) and how each page “looks” at a block level.
Homepage / Market dashboard
* Purpose: “market at a glance” + entry points into indices, signals, and tickers.
* Looks like: hero tagline + market snapshot widgets + heatmap preview + “Top signals today” preview + performance proof preview + navigation to core sections.

o   Looks like: search + filters + sortable table + pagination.

Indices

Specific SEO elements per page
This section focuses strictly on the required SEO elements per page: human-readable URLs, H1, meta title, meta description, plus the content logic that makes those elements consistent and scalable across hundreds of programmatic pages.
Global rules for URL patterns
Use descriptive, readable URL paths and keep the structure logical (directory grouping by page type). This aligns with Google guidance on using descriptive URLs and structuring content so URLs are intelligible to humans and crawlers.
Avoid generating large volumes of near-duplicate URLs from filters and layered parameters; Google specifically warns that overly complex URLs and additive filtering can create many redundant URLs and cause crawl/indexing inefficiency.

Examples grounded in Odin500 data and competitor patterns
Competitor patterns to copy in MVP-friendly form
Competitors that strongly resemble Odin500’s intended “terminal + screener + visual map” experience consistently emphasize: (1) powerful chart/technical views, (2) heatmaps and scanning workflows, (3) watchlists/dashboards, and (4) credible “research / performance proof” blocks.
TradingView strongly emphasizes advanced charts, technical analysis tooling, screeners, heatmaps, and alerts as core product surfaces; these are useful UX patterns for Odin500’s Signals hub and Ticker detail charts.
Finviz explicitly documents “market maps” as a key visualization for browsing and analyzing large amounts of stock data; this matches Odin500’s index heatmap requirement.
Koyfin positions customizable dashboards and powerful watchlists as primary surfaces; this maps cleanly onto Odin500’s Homepage/Dashboard and user “universes/watchlists” theme.
TrendSpider highlights automated technical analysis, multi‑timeframe analysis, indicators, and pattern recognition—useful as “Phase 2/3” inspiration rather than MVP requirements.
TipRanks structures stock pages around forecasts, analyst ratings, and multiple research sections (technical analysis, historical prices, statistics). Even if Odin500 doesn’t replicate analyst data, the “report/storytelling” layout is a useful model for Odin500’s Reports pages.
